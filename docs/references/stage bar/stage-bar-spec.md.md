# references/stage bar/stage-bar-spec.md

**Path & purpose** — `references/stage bar/stage-bar-spec.md`. A detailed, prescriptive implementation handoff spec for a UI feature: replacing the match HUD's old round-dots banner + stage-caption/radial-dial cluster with one unified "stage bar" component (`renderStageBar`) showing a stage marker, a 7-icon round schedule strip, and a countdown timer + progress bar, all top-center over the board.

**Responsibility** — A point-in-time design/implementation brief (not living documentation, not source code) written to hand off a specific visual feature to a coder agent. It specifies exact pixel geometry, color constants, glyph designs, and code-replacement instructions for `packages/client/src/scenes/match.ts`, `packages/client/src/glyphs.ts`, and `packages/client/src/theme.ts`.

**Key behavior — what the spec describes**
- **Component**: `renderStageBar(centerX, statusY, state)` — a trapezoidal teal-glass panel (184px wide bottom edge, 198px top edge via a 7px outward slant, 22px tall) centered horizontally over the board, anchored to the top of the status row. Three internal zones: Zone A (stage marker: `"banner"` glyph + bold "stage-round" text e.g. "6-2"), Zone B (round schedule strip: one glyph per round in the current stage — `"swords"` for PvP, `"gem"` for non-final PvE, `"monster"` for the stage's final/boss PvE round — with the CURRENT round's icon enlarged, gold-colored, and marked with a small upward chevron), Zone C (timer: `"clock"` glyph + bold seconds count, tinted `C.hpLow` under 5 seconds). A separate 4px-tall progress-bar capsule sits 3px beneath the trapezoid, depleting left-to-right as `timeLeft/PLANNING_TIMER_MS` shrinks.
- **New glyphs specified**: `"clock"` (circle + two hands), `"monster"` (a filled bat-silhouette poly, deliberately distinct from the `undead` trait's skull glyph to avoid meaning-collision), `"banner"` (pole + small triangular pennant) — all to be added to `glyphs.ts`'s `GlyphKind` union and `drawGlyph` switch, explicitly NOT added to `TRAIT_GLYPH` (non-trait HUD icons).
- **New theme constants specified**: `stageBarBg` (0x16302c), `stageBarBorder` (0xc9a24a, deliberately reusing the `itemFrame` gold hue rather than `accentGold`, to preserve the project's "two golds" split), `stageBarTrack` (0x0a1210), `stageProgress` (0x4fd6c8, deliberately distinct from `xpArcFill` despite similar hue, for independent retunability).
- **Round→glyph mapping**: derived purely from existing rules exports (`stageForRound`, `isPveRound`, `STAGE1_LEN`/`STAGE_LEN`) — the spec is explicit that "no new game logic" is introduced; every value the bar renders is a pass-through of values the codebase already computes.
- **Replacement instructions**: deletes `renderRoundDots` and the old stage-caption/radial-dial block inside `renderHud`; repositions `renderSkipButton` beneath the new bar+progress-bar stack via new helpers `stageBarTopY`/`stageBarBottomY`; notes the landscape layout needs no separate variant (same fixed 184px width, centered over `regions.board`).
- Ends with an explicit handoff line: "→ coder: implement `renderStageBar`... → state-scribe: record spec completion," confirming this document's role as a one-shot work-order in an agent-driven development workflow (a "coder" subagent receives this file as its complete spec, then a "state-scribe" agent/process is expected to record that the spec was fulfilled).
